# Bank-Customer-Churn-Prediction
Predicting Customer Сhurn
 # Прогнизирование оттока клиентов банка
> ML-проект, направленный на снижение оттока клиентов через раннее выявление рискованных профилей.  
> **Бизнес-ценность**: удержание 1 клиента = ~2000 ₽. При охвате 1000 клиентов в месяц — потенциальная экономия **2 млн ₽/год**.

## Проблема
Банки теряют клиентов из-за отсутствия проактивных мер удержания. Прогнозирование оттока за 1–2 месяца позволяет запустить персональные предложения (cashback, снижение ставки и т.п.).

## Данные

- Источник: [Kaggle — Bank Customer Churn Prediction](https://www.kaggle.com/datasets/shantanudhakadd/bank-customer-churn-prediction)  
- 15 000 клиентов
- Признаки: возраст, баланс, количество продуктов, активность, страна, пол, кредитная карта и др.
- Целевая переменная: `Exited` (1 — ушёл, 0 — остался)

## Подход
1. **Очистка данных**: обработка пропусков, выбросов, кодирование категорий.
2. **Feature Engineering**:  
   - Создан признак `BalancePerProduct` (баланс / кол-во продуктов)  
   - Добавлены логические флаги (`HasHighBalance`, `IsActiveAndOld`)  
3. **Моделирование**:  
   - Протестированы Logistic Regression, Random Forest, XGBoost  
   - Лучшая модель: **XGBoost**

4. **Интерпретация**: важнейшие признаки — `Balance`, `IsActiveMember`, `NumOfProducts`

## Бизнес-рекомендации
- Целиться на клиентов с **низкой активностью + высоким балансом** — они чаще уходят.
